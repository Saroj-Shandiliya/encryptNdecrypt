About


Encrypt n Decrypt

is all about Freedom. We here at EnD value freedom and control over things that are ours. As privacy over internet is a greater concern in todays generation and data privacy is a matter that we all talk about. We here at EnD proudly provide a messaging platform that lets the users control over the messages they send.And these messages are encrypted in a way that they can be decrypted only once and so once these messages are decrypted they cannot be decrypted again and so these encrypted messages once decrypted becomes nothing more than just some random text.

How does it work?

Once messages are created from the EnD platform these messages are encrypted via an algorithm these algorithm encrypts these messages in a way thet they can only be decrypted once. Once the message is decrypted those encrypted message remains of no use to anyone and is just a peice of random text.

How to use EnD?

Once a user joins the EnD family the user has various methods in which he/she can create messages.
To create message the user goes the navigation menu and selects the encrypt menu ,the user is then prompted with a create message form the users type the message there and then select the type of message the user wants to create:

1.User ID(Direct):

In this method the user can send message to another EnD family member using their registered email id and once created the other member can decrypt that message.

2.User ID(via Key):

This method is similar to the first method just the difference is that in this method the messages are created using a secure key whih the other member knows about as before decrypting that message it will be required.

3.Guest ID:

In this method a member can create a message using a secure key not for any other member but a guest someone who is not a member of EnD family a Guest ID is given once the message is created and the Geust ID is to be entered in the Guest ID page with the key to acess that message.(These message can be viewed only once)And once viewed the Guest ID gets expired.

4.Random

In this method a member creates a message without a secure key for random people or a group of people. This message is available of a day and gets expired once the day ends midnight.Once the message is created a random encrypted message is generated which can be used to enter the random room and see the random message.
